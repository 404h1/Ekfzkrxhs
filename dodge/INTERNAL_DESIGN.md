# Zero Trace 내부 기획 문서

이 문서는 개발/기획 내부 공유용 문서다. 모든 스포일러를 포함한다.

## 한 줄 설명

겉으로는 60초 생존 탄막 게임처럼 보이지만, 반복 플레이를 통해 각종 만우절식 배신 패턴과 가짜 최종 엔딩 구조가 드러나는 장난형 게임이다.

## 핵심 목표

- 첫인상은 정상적인 실력형 게임처럼 보여야 한다.
- 장난의 정체가 너무 빨리 들키면 안 된다.
- 억까는 즉시 사기보다 "이제 되겠다" 싶은 순간 뒤집혀야 한다.
- 유저가 짜증보다 헛웃음을 먼저 내게 만드는 것이 목표다.

## 플레이 구조

- 기본 목표: 60초 생존
- 실제 구조: 루트별로 다른 방식의 실패 연출을 경험하고 도감에 기록을 쌓는다.
- 수집 구조: 실제 해금 가능한 엔딩은 12개
- 숨은 장치: `END 99`는 존재하는 것처럼 보이지만 실제로는 해금 불가능

## 루트 순서

플레이 순서는 아래 순환 구조를 따른다.

1. `normal`
2. `instant`
3. `hope`
4. `pattern`
5. `growth`
6. `reward`
7. `hudDrop`
8. `drift`
9. `darkness`
10. `wall`
11. `bigBullet`
12. `error59`

첫 도전은 `normal`부터 시작한다.

## 루트 상세

### normal

- 역할: 기준점
- 인상: 완전히 정상적인 탄막 생존 게임
- 목적: 유저가 게임의 기본 법칙을 믿게 만든다.
- 도감 엔딩: `END 00`

### instant

- 역할: 충격용 즉사
- 초반: 시작 직후 짧은 정적
- 결말: 사실상 피할 수 없는 첫 탄막으로 바로 종료
- 웃음 포인트: 플레이보다 엔딩 수집이라는 정체를 너무 빠르게 이해하게 됨
- 도감 엔딩: `END 01`

### hope

- 역할: 희망 착각
- 초반: 다른 루트보다 안정적이라 자신감이 생김
- 중반: "이 흐름이면 된다"는 감각 제공
- 결말: 마지막 한 번의 닫힘으로 안전 공간 회수
- 웃음 포인트: 가능하다고 믿은 순간 정확히 그 믿음 때문에 죽음
- 도감 엔딩: `END 02`

### pattern

- 역할: 학습 배신
- 초반: 같은 패턴이 반복됨
- 중반: 외우면 된다는 확신 형성
- 결말: 마지막 반복에서 안전 구간이 한 칸 틀어짐
- 웃음 포인트: 내가 틀린 건지 게임이 틀린 건지 헷갈림
- 도감 엔딩: `END 03`

### growth

- 역할: 후반 판정 배신
- 초반: 정상 플레이
- 중반: 기체 외곽이 미세하게 커지기 시작
- 후반: 충돌 판정이 급격히 커져 사실상 안전 구간이 무너짐
- 웃음 포인트: "내가 못 피한 게 아니라 몸이 너무 커졌네?"
- 도감 엔딩: `END 04`

### hudDrop

- 역할: HUD 반란
- 초반: 비교적 정상 루트처럼 진행
- 중반: 상단 HUD pill이 분리돼 하나씩 낙하
- 후반: 두 개씩 같이 쏟아지며 실제 장애물이 됨
- 웃음 포인트: 시간/레벨 UI가 설명이 아니라 물리 오브젝트가 됨
- 도감 엔딩: `END 11`

### drift

- 역할: 입력 신뢰 붕괴
- 초반: 정상 조작
- 중반: 입력 오차가 누적되며 포인터와 기체가 어긋남
- 후반: 기준점이 멋대로 틀어져 조작감이 무너짐
- 웃음 포인트: "왜 내가 저기로 갔지?"가 반복되며 시스템을 의심하게 됨
- 도감 엔딩: `END 07`

### darkness

- 역할: 희망 제공 후 재배신
- 초반: 점진적 암전
- 중간: 강제 일시정지 + 화면 밝기 최대로 올릴 기회 제공
- 재개 후: 밝기를 올렸는데도 어둠이 더 짙고 빠르게 진행
- 웃음 포인트: 친절한 척했는데 결국 더 심해짐
- 구현 주의: 브라우저는 기기 밝기를 직접 조절할 수 없으므로 모달 안내 방식 사용
- 도감 엔딩: `END 08`

### wall

- 역할: 보이는 희망 닫기
- 초반: 정렬된 장벽 패턴
- 핵심: 통로가 열려 보이지만 진입 직전에 그 틈을 막고, 빠져나와도 한 번 더 닫힘
- 웃음 포인트: "분명 들어갈 틈이 있었는데?"
- 도감 엔딩: `END 09`

### reward

- 역할: 시스템 창 방해
- 초반: 비교적 정상 루트처럼 진행
- 중반: 시스템 오류창처럼 보이는 거대한 패널 출력
- 후반: 오류창이 시야를 막는 동안 탄막은 계속 진행
- 웃음 포인트: 게임이 고장난 척하면서 더 정확하게 죽임
- 도감 엔딩: `END 06`

### bigBullet

- 역할: 상식 밖 과장
- 초반: 조금 빠른 일반 루트처럼 진행
- 중반: 대형 반응 경고
- 후반: 큰 탄환이 등장한 뒤 멈추지 않고 계속 커짐
- 웃음 포인트: "저건 너무 큰데, 왜 계속 더 커져?"
- 도감 엔딩: `END 10`

### error59

- 역할: 시간 단축 배신
- 초반~중반: 이번 판만 30초 특별 세션처럼 보이고 실제 난이도도 매우 쉬워 안심을 준다
- 후반: 타이머가 29초 부근에서 미세하게 이상해짐
- 결말: 29초대에서 예외 처리처럼 종료
- 웃음 포인트: 시간을 줄여준 줄 알았는데 그 짧은 세션조차 함정이었다는 점
- 도감 엔딩: `END 05`

## 도감 구조

### 실제 수집 엔딩

일반 충돌은 `END 00` 기본 기록으로 귀속된다. 각 루트의 고유 기믹이 발동한 뒤 종료될 때만 해당 특수 엔딩이 열린다.

| 엔딩 ID | 내부 키 | 의미 |
| --- | --- | --- |
| END 00 | `normalFail` | 기본 패턴 종료 |
| END 01 | `instant` | 시작 직후 즉사 루트 |
| END 02 | `hope` | 희망 착각 루트 |
| END 03 | `pattern` | 패턴 학습 배신 루트 |
| END 04 | `growth` | 플레이어 판정 확대 루트 |
| END 05 | `error59` | 29초 시간 단축 함정 루트 |
| END 06 | `reward` | 시스템 오류창 방해 루트 |
| END 11 | `hudDrop` | 상단 HUD 낙하 루트 |
| END 07 | `drift` | 입력 오차로 조작이 어긋나는 루트 |
| END 08 | `darkness` | 암전 + 밝기조정 배신 루트 |
| END 09 | `wall` | 열린 틈이 닫히는 장벽 루트 |
| END 10 | `bigBullet` | 대형 탄환 팽창 루트 |

상단 카운트는 `0 / 12`로 표시되고, 위 12개만 집계한다.

### 가짜 최종 엔딩

- 엔딩 ID: `END 99`
- 이름: `완전 생존`
- 도감 노출: 처음부터 최종 엔딩처럼 보이게 노출
- 표시 조건: "모든 기록 수집 후 무피격 60초 생존"
- 실제 상태: 절대 해금되지 않음
- 역할: 유저가 끝까지 진엔딩을 기대하게 만드는 UI 속임수

## UX 원칙

- HUD는 장난의 정답을 먼저 말하면 안 된다.
- 초반 5초 안에 루트 정체가 다 들키면 실패다.
- 도감은 수집 동기를 만들어야 한다.
- 결과 화면은 짜증보다 "아, 당했다"는 정리를 제공해야 한다.

## 체크리스트

- 첫인상이 정상 게임처럼 보이는가
- 각 루트가 너무 빨리 스포일러되지 않는가
- 배신 타이밍이 늦고 선명한가
- `END 99`가 여전히 진짜 보상처럼 보이는가
- 수집 카운트가 `END 99`를 제외한 12개만 세고 있는가
