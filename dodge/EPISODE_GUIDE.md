# Zero Trace 에피소드 / 도감 설명서

이 문서는 `dodge_game.html` 기준으로 현재 게임의 에피소드 구성, 도감 구조, 연출 의도를 정리한 내부용 설명서다.

## 게임 개요

- 장르: 60초 생존형 탄막 회피 게임
- 기본 인상: 처음에는 정상적인 아케이드 생존 게임처럼 보이지만, 진행할수록 만우절식 배신 연출이 드러난다.
- 핵심 감정선: "이번에는 될 것 같은데?"에서 시작해서 "아니 이건 뭐야?"로 넘어가며 웃게 만드는 구조
- 기록 구조: 각 실패 패턴이 도감 엔딩으로 누적 저장된다.

## 루트 순서

도전 순서에 따라 아래 루트가 순환한다.

1. `normal`
2. `instant`
3. `hope`
4. `pattern`
5. `growth`
6. `reward`
7. `hudDrop`
8. `drift`
9. `darkness`
10. `wall`
11. `bigBullet`
12. `error59`

첫 플레이는 반드시 `normal`부터 시작한다.

## 에피소드 설명

### 1. normal

- 역할: 기준점 역할을 하는 기본 루트
- 인상: 가장 정상적인 탄막 게임처럼 보인다.
- 의도: 유저가 "이 게임은 실력으로 버티는 게임"이라고 믿게 만드는 시작점
- 결과 엔딩: `END 00`

### 2. instant

- 역할: 충격용 즉사 루트
- 초반: 시작 직후 아주 짧은 정적
- 결말: 회피를 시작하기도 전에 첫 탄막이 사실상 끝을 낸다.
- 의도: 수집 게임이라는 정체를 빠르게 각인시키는 첫 장난
- 결과 엔딩: `END 01`

### 3. hope

- 역할: 가능성 착각 루트
- 초반: 다른 루트보다 안정적으로 느껴진다.
- 중반: "이 흐름이면 된다" 싶은 안정감이 생긴다.
- 결말: 안전해 보이던 공간이 마지막 한 번의 닫힘으로 무너진다.
- 의도: 실력으로 버텼다고 믿게 만든 뒤 희망만 정확히 회수하는 것
- 결과 엔딩: `END 02`

### 4. pattern

- 역할: 학습 배신 루트
- 초반: 반복되는 동일 패턴이 보인다.
- 중반: 몇 차례 같은 형태가 반복되어 외우면 될 것처럼 느껴진다.
- 결말: 마지막 반복에서 딱 한 번, 안전 구간이 옆으로 틀어진다.
- 의도: "이제 알겠다"는 순간 틀어 버리는 악질 설계
- 결과 엔딩: `END 03`

### 5. growth

- 역할: 후반 배신형 억까 루트
- 초반: 일반 패턴처럼 보이며 판정도 정상이다.
- 중반: 기체 외곽이 조금씩 커지기 시작한다.
- 후반: 충돌 판정이 급격히 커져 안전 구간이 사실상 무너진다.
- 의도: 뒤늦게 "왜 나만 커져?"라는 느낌을 주는 것
- 결과 엔딩: `END 04`

### 6. reward

- 역할: 시스템 창 방해 루트
- 초반: 일반 루트처럼 진행된다.
- 중반: 시스템 오류창처럼 보이는 대형 패널이 전투 중앙에 뜬다.
- 후반: 창이 시야를 가리는 동안 탄막은 멈추지 않는다.
- 의도: 플레이어를 돕는 척하는 UI가 실제론 가장 큰 방해물이 되는 것
- 결과 엔딩: `END 06`

### 7. hudDrop

- 역할: 상단 HUD 낙하 루트
- 초반: 평범한 탄막처럼 진행된다.
- 중반: 시간, 레벨, 상태, 힌트 pill이 위에서 하나씩 떨어지기 시작한다.
- 후반: 낙하 속도가 빨라지고 두 개씩 동시에 떨어진다.
- 의도: 플레이어를 돕던 UI가 그대로 장애물로 변하는 어이없는 배신
- 결과 엔딩: `END 11`

### 8. drift

- 역할: 입력 신뢰 붕괴 루트
- 초반: 정상 조작
- 중반: 입력 오차가 누적되며 포인터와 기체 위치가 서서히 어긋나기 시작한다.
- 후반: 보정 기준점이 멋대로 움직여 조작감이 무너진다.
- 의도: 자기 실수인지 시스템 문제인지 잠깐 헷갈리게 만드는 것
- 결과 엔딩: `END 07`

### 9. darkness

- 역할: 희망을 줬다가 다시 빼앗는 루트
- 초반: 화면이 점차 어두워진다.
- 중간: 게임이 강제로 일시정지되고, 유저에게 디바이스 화면 밝기를 최대로 올릴 기회를 준다.
- 재개: 밝기를 올렸는데도 다시 더 짙은 어둠이 내려온다.
- 의도: "그래도 배려해 주네?"라고 생각하게 만든 뒤 다시 배신하는 구조
- 결과 엔딩: `END 08`

### 10. wall

- 역할: 안전지대 낚시 루트
- 초반: 정렬된 장벽 패턴이 준비된다.
- 핵심: 통과 가능한 틈이 보이도록 만든 뒤, 진입 타이밍에 그 틈을 닫고 운 좋게 빠져도 한 번 더 덮친다.
- 의도: "분명 안전했는데?"라는 허탈한 웃음을 유도하는 것
- 결과 엔딩: `END 09`

### 11. bigBullet

- 역할: 상식 밖 과장 루트
- 초반: 조금 빠른 일반 패턴처럼 보인다.
- 중반: 대형 객체 경고가 뜬다.
- 후반: 탄환이 화면 안에서 계속 비정상적으로 커진다.
- 의도: 즉사 사기보다 "저게 왜 계속 커져?"를 먼저 느끼게 만드는 것
- 결과 엔딩: `END 10`

### 12. error59

- 역할: 시간 단축 루트
- 초반~중반: 매우 쉬운 생존 게임처럼 진행된다.
- 초반: 이번 판만 30초 특별 세션처럼 보인다.
- 후반: 타이머가 29초 부근에서 미묘하게 이상해지기 시작한다.
- 결말: 29초대에서 시간이 거의 멈춘 것처럼 끝난다.
- 의도: 시간이 줄어들어 안심한 순간 장난의 정체가 드러나게 하는 것
- 결과 엔딩: `END 05`

## 도감 구조

### 실제 수집 엔딩

실제로 해금 가능한 엔딩은 아래 12개다.

초반 일반 충돌은 `END 00` 기본 기록으로 처리되며, 각 루트의 고유 장난이 실제로 발동한 뒤 종료되어야만 특수 기록이 열린다.

| 엔딩 ID | 내부 키 | 의미 |
| --- | --- | --- |
| END 00 | `normalFail` | 정상적인 탄막 플레이 패배 |
| END 01 | `instant` | 시작 직후 즉사 루트 |
| END 02 | `hope` | 희망을 줬다가 닫아 버리는 루트 |
| END 03 | `pattern` | 반복 학습을 배신하는 루트 |
| END 04 | `growth` | 플레이어 판정 확대 루트 |
| END 05 | `error59` | 29초 시간 단축 함정 루트 |
| END 06 | `reward` | 시스템 오류창 방해 루트 |
| END 11 | `hudDrop` | 상단 HUD 낙하 루트 |
| END 07 | `drift` | 입력 오차로 조작이 어긋나는 루트 |
| END 08 | `darkness` | 어둠과 밝기 조정 배신 루트 |
| END 09 | `wall` | 열렸다가 닫히는 장벽 루트 |
| END 10 | `bigBullet` | 비정상적으로 커지는 대형 탄환 루트 |

상단 수집 카운트는 `0 / 12` 형태로 표시되며, 이 12개만 집계에 포함된다.

### 가짜 최종 엔딩

- 엔딩 ID: `END 99`
- 이름: `완전 생존`
- 도감 표시: 처음부터 최종 엔딩 슬롯처럼 보인다.
- 표시 조건 문구: "모든 기록 수집 후 무피격 60초 생존"
- 실제 상태: 해금 불가능
- 의도: "모든 엔딩을 모으면 진엔딩이 있겠지"라는 기대를 만들기 위한 UI 속임수
- 중요: 이 엔딩은 도감 카드에는 보이지만 실제 수집 수에는 포함되지 않는다.

## 도감 UX

- `기록 보관함` 버튼으로 언제든 열 수 있다.
- `닫기` 버튼으로 닫을 수 있다.
- 모달 바깥 배경을 클릭해도 닫힌다.
- `Esc` 키를 눌러도 닫힌다.
- 해금된 엔딩은 브라우저 `localStorage`에 자동 저장된다.

## 연출 원칙

- 초반에는 설명을 줄이고 정상적인 게임처럼 보여야 한다.
- 장난은 너무 빨리 들키면 안 된다.
- 억까는 무조건 즉사보다 "희망을 보여 준 뒤 뒤집는 방식"이 더 재밌다.
- 결과 화면과 도감은 유저가 "당했다"는 사실을 정리해 주는 역할을 한다.
- `END 99`는 진짜 보상처럼 보여야 하지만 실제로는 절대 도달할 수 없어야 한다.

## 수정할 때 체크할 기준

- 이 루트가 첫 5초 안에 정체를 너무 빨리 드러내는가
- HUD 문구가 장난의 정답을 먼저 말해 버리지는 않는가
- 유저가 "이건 너무 억지"보다 "아 진짜 웃기네" 쪽으로 반응할 수 있는가
- 도감에서 `END 99`가 여전히 진짜 최종 엔딩처럼 보이는가
- 수집 카운트가 여전히 `END 99`를 제외한 12개만 세고 있는가
